- 页面布局不要铺满屏幕
- 不要过度工程化

## 三、整体风格与 UI 规范

### 视觉风格
- 卡片式组件风格
- 低饱和度配色
- 东方美学（太极、五行、日月）
- 克制、安静、有深度
- 圆角 + 柔和阴影
- 呼吸感留白
- 动态但克制

### 布局规则
- 页面内容居中，自适应宽度
- 左侧为主交互区域
- 右侧为信息流展示区域（竖向自动滚动）
- 整体页面不铺满视口宽度

### 全局功能
- 页面右上角提供主题色切换按钮
- 主题状态使用 `localStorage` 保存
- 首次进入页面时自动播放轻音乐（允许用户关闭）

#### （1.1）Terminal 命令行模拟模块

界面要求：
- 终端风格卡片
- 深色背景
- 等宽字体

支持命令：
```

help
ls
cd flame
cd flumina
cd webgame

```

行为逻辑：
- `ls`：列出当前已部署的网站（使用简写名）
- `cd xxx`：浏览器跳转至对应网站

命令映射关系：
- flame → https://sunyzhi55.github.io/astro-firefly-blog/
- flumina → https://sunyzhi55.github.io/vitepress-teek-blog/
- webgame → https://sunyzhi55.github.io/WebGame/

说明：
- 该模块为前端命令模拟，并非真实 shell

---

#### （1.2）太极 · 五行交互融合系统

核心视觉：
- 页面中央为持续旋转的太极 / 五行圆盘
- 周围分布五个可点击节点：
  - 金、木、水、火、土
- 日 / 月作为阴阳调节因子存在

交互逻辑：
- 元素支持多选
- 选中状态有明显视觉反馈
- 点击【熔合】按钮后生成新的抽象视觉效果

融合展示规则（艺术化表达）：
- 火 + 木 → 火焰藤蔓
- 水 + 金 → 液态金属
- 土 + 火 → 熔岩岩石
- 日 + 火 → 白焰
- 月 + 水 → 冷光流体

技术实现建议：
- 使用 SVG 或 Canvas
- 动画使用 CSS 或轻量 JS
- 使用 Astro 的 `client:only` 组件实现
